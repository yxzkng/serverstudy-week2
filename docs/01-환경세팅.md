# 01 — 환경 세팅 (모임 전 필수 예습)

> **이 문서는 모임 전에 반드시 끝내둔다.** 

---

## 1. 준비물

- **Docker Desktop** — 컨테이너를 실행하는 프로그램
- **1주차에서 clone한 spring-petclinic 폴더** — 없다면 아래로 새로 받는다
  ```bash
  git clone https://github.com/spring-projects/spring-petclinic.git
  cd spring-petclinic
  ```
- (지난주에 설치한) **JDK 17 이상** — 이미지 빌드에 필요

---

## 2. Docker Desktop 설치

1. https://www.docker.com/ 접속 → 자신의 OS에 맞는 **Docker Desktop** 다운로드
2. 설치 후 실행 → 좌하단(또는 상단 고래 아이콘)에 **"Engine running"** 또는 초록불이 뜨면 준비 완료
3. Windows는 최초 실행 시 WSL2 설치를 요구할 수 있다. 안내에 따라 설치하고 재부팅한다.

> **Apple Silicon Mac(M1~M4) 사용자 참고**: 실행 중 "Rosetta installation failed" 경고가 뜰 수 있다.
> **`Disable Rosetta`를 눌러도 이번 실습엔 지장이 없다.** (자세한 건 트러블슈팅 문서 참고)

---

## 3. 설치 확인

터미널에서 아래 두 명령이 정상 출력되면 성공이다.

```bash
docker --version
docker info
```

- `docker --version` → `Docker version 2x.x.x, build ...` 처럼 버전이 나온다.
- `docker info` → 서버(daemon) 정보가 쭉 나온다. 만약 `Cannot connect to the Docker daemon` 같은 오류가 나면 **Docker Desktop이 안 켜진 것**이다. Docker Desktop을 먼저 실행한다.

---

## 4. (중요) 이미지 미리 빌드해두기

### 4-1. 먼저 build.gradle 수정 — 이거 안 하면 빌드가 실패한다 ⚠️

최신 PetClinic(Spring Boot 4.x)은 **GraalVM 네이티브 이미지** 빌드 설정을 포함하고 있다. 그대로 `bootBuildImage`를 실행하면 우리가 원하는 일반 JVM 이미지가 아니라 네이티브 빌드를 시도하다가 **실패한다**(`Builder lifecycle 'creator' failed with status code 51`). spring-petclinic 폴더의 `build.gradle`에서 **네이티브 관련 8줄을 주석 처리**한다. (버전 숫자는 클론 시점에 따라 다를 수 있다)

```groovy
plugins {
  ...
  // id 'org.graalvm.buildtools.native' version '1.1.2'    ← ① 이 줄 주석
  ...
}

// ② 이 줄 주석 (파일 상단부)
// gradle.startParameter.excludedTaskNames += [ "checkFormatAot", "checkFormatAotTest" ]

// ③ 파일 맨 아래 6줄도 전부 주석
// checkstyleAot.enabled = false
// checkstyleAotTest.enabled = false
// checkFormatAot.enabled = false
// checkFormatAotTest.enabled = false
// formatAot.enabled = false
// formatAotTest.enabled = false
```

> **왜?** 네이티브 이미지는 자바 앱을 기계어로 미리 컴파일하는 별도 기술이다(시작이 빠르고 메모리를 적게 쓰는 대신 빌드가 까다롭다). 이번 주 목표는 "JVM 앱을 컨테이너로 운영하기"이므로 꺼둔다. 참고로 ①만 주석하면 ②③ 줄이 이제는 존재하지 않는 Aot 태스크를 참조하게 되어 다른 에러가 나니, **8줄을 모두** 주석해야 한다.

### 4-2. 이미지 빌드

PetClinic 폴더 안에서 아래를 실행한다. **spring-petclinic 폴더 안에서** 실행해야 한다.

```bash
./gradlew bootBuildImage --imageName=spring-petclinic:1.0
# Windows(PowerShell): .\gradlew.bat bootBuildImage --imageName=spring-petclinic:1.0
```

- **처음엔 오래 걸린다(수 분).** 빌드에 필요한 베이스 이미지를 인터넷에서 여러 개 받기 때문이다. 커피 한 잔 하고 오면 된다.
- 끝나면 `BUILD SUCCESSFUL`이 뜬다.
- 확인:
  ```bash
  docker images | grep petclinic
  ```
  `spring-petclinic   1.0   ...` 한 줄이 보이면 성공이다. (Windows PowerShell은 `grep` 대신 `findstr`)

> **왜 Dockerfile이 없는데 이미지가 만들어지나?**
> PetClinic엔 Dockerfile이 없다. `bootBuildImage`가 Cloud Native Buildpacks를 써서 자바 앱용 이미지를 자동 생성한다. 원리는 `02-이론.md`에서 다룬다.

---

## 5. 여기까지 되면 준비 끝

- `docker --version` / `docker info` OK
- `docker images`에 `spring-petclinic:1.0` 보임

이 두 가지가 확인되면 모임 준비는 완료다. 이론 문서(`02-이론.md`)를 읽고 오면 실습이 훨씬 수월하다.

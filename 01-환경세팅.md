# 01 — 환경 세팅 (모임 전 필수 예습)

> **이 문서는 모임 전에 반드시 끝내둔다.** 특히 마지막 "이미지 미리 빌드"는
> 처음 실행 시 수 분이 걸리므로, 당일에 하면 다 같이 기다리게 된다.

---

## 1. 준비물

- **Docker Desktop** — 컨테이너를 실행하는 프로그램
- **1주차에서 clone한 spring-petclinic 폴더** — 없다면 아래로 새로 받는다
  ```powershell
  git clone https://github.com/spring-projects/spring-petclinic.git
  cd spring-petclinic
  ```
- (지난주에 설치한) **JDK 17 이상** — 이미지 빌드에 필요

---

## 2. Docker Desktop 설치

1. https://www.docker.com/ 접속 → 자신의 OS에 맞는 **Docker Desktop** 다운로드
2. 설치 후 실행 → 좌하단(또는 상단 고래 아이콘)에 **"Engine running"** 또는 초록불이 뜨면 준비 완료
3. Windows는 최초 실행 시 WSL2 설치를 요구할 수 있다. 안내에 따라 설치하고 재부팅한다.

> **Apple Silicon Mac(M1~M4) 사용자 참고**: 실행 중 "Rosetta installation failed" 경고가 뜰 수 있다.
> **`Disable Rosetta`를 눌러도 이번 실습엔 지장이 없다.** (자세한 건 트러블슈팅 문서 참고)

---

## 3. 설치 확인

터미널(PowerShell)에서 아래 두 명령이 정상 출력되면 성공이다.

```powershell
docker --version
docker info
```

- `docker --version` → `Docker version 2x.x.x, build ...` 처럼 버전이 나온다.
- `docker info` → 서버(daemon) 정보가 쭉 나온다. 만약 `Cannot connect to the Docker daemon` 같은 오류가 나면 **Docker Desktop이 안 켜진 것**이다. Docker Desktop을 먼저 실행한다.

---

## 4. (중요) 이미지 미리 빌드해두기

PetClinic 폴더 안에서 아래를 실행한다. **spring-petclinic 폴더 안에서** 실행해야 한다.

```powershell
.\gradlew.bat bootBuildImage --imageName=spring-petclinic:1.0
```

- **처음엔 오래 걸린다(수 분).** 빌드에 필요한 베이스 이미지를 인터넷에서 여러 개 받기 때문이다. 커피 한 잔 하고 오면 된다.
- 끝나면 `BUILD SUCCESSFUL`이 뜬다.
- 확인:
  ```powershell
  docker images | findstr petclinic
  ```
  `spring-petclinic   1.0   ...` 한 줄이 보이면 성공이다. (macOS/Linux는 `findstr` 대신 `grep`)

> **왜 Dockerfile이 없는데 이미지가 만들어지나?**
> PetClinic엔 Dockerfile이 없다. `bootBuildImage`가 Cloud Native Buildpacks를 써서 자바 앱용 이미지를 자동 생성한다. 원리는 `docs/02-이론.md`에서 다룬다.

---

## 5. 여기까지 되면 준비 끝

- `docker --version` / `docker info` OK
- `docker images`에 `spring-petclinic:1.0` 보임

이 두 가지가 확인되면 모임 준비는 완료다. 이론 문서(`02-이론.md`)를 읽고 오면 실습이 훨씬 수월하다.
